● Why Do We Need “Nonlinear” Dimensionality Reduction?

This section introduces the key conceptual foundation of dimensionality reduction. In single-cell analysis, we routinely work with extremely high-dimensional data. Understanding why linear methods alone are not enough is essential before moving on to nonlinear algorithms.

Limitations of Traditional PCA (Linear)

Principal component analysis (PCA) is a standard preprocessing tool, but it mainly captures global variance in the data. When cellular structure follows a complex biological manifold, PCA often struggles to separate subtle subpopulations and may leave biologically distinct cells overlapped in low-dimensional space.

Advantages of Nonlinear Dimensionality Reduction

  • Preserves local neighborhood relationships between cells in high-dimensional space.
  • Can “unfold” complex nonlinear structures into a two-dimensional map, making cellular clusters and biological trajectories much easier to interpret visually.

Comparison of Key Parameters

These parameters directly influence how tightly cells are grouped and how distances between clusters appear in the final embedding. Understanding them is essential for optimizing the visualization.

Parameter t-SNE UMAP
Core variables Perplexity n_neighbors / min_dist
What they influence Controls the balance between local and broader structure by effectively adjusting how many neighbors are considered. n_neighbors controls local structure; min_dist controls how tightly points are packed in the embedding.

Practical Workflow (Seurat in R)

Below is a standard analysis workflow. Note that before running nonlinear dimensionality reduction, you must first run PCA to denoise the data and improve computational efficiency. 

# Assume normalization and highly variable gene selection are already done
# 1. Run PCA (linear dimensionality reduction)
pbmc <- RunPCA(pbmc, features = VariableFeatures(object = pbmc))

# 2. Choose the number of principal components (for example, the first 20 PCs)
# 3. Run UMAP (recommended)
pbmc <- RunUMAP(pbmc, dims = 1:20)

# 4. Run t-SNE (optional)
pbmc <- RunTSNE(pbmc, dims = 1:20)

# 5. Visualize
DimPlot(pbmc, reduction = "umap", label = TRUE)

Practical Summary

Use t-SNE When the main goal is to separate and display clearly defined, discrete cell types.
Use UMAP (Recommended) When the data include developmental trajectories, continuous cellular states, or very large numbers of cells.

Visualization should be treated as a tool for interpretation, not as the final proof. Biological conclusions must ultimately rely on the gene expression matrix and independent experimental validation.
