Identifying Highly Variable Genes
(Feature Selection)

In single-cell analysis, not all genes are equally informative. This guide walks you through how feature selection identifies the key genes that vary meaningfully across cell populations, providing a strong foundation for downstream dimensionality reduction and clustering.

1. Why Do We Need Feature Selection?

This section introduces the three main purposes of feature selection in single-cell data analysis. By removing uninformative features, we can focus on the biological signals that truly define cellular diversity. Hover over the cards below to explore each concept.

⬋

Overcoming the Curse of Dimensionality

Single-cell datasets often contain more than 20,000 genes, but most genes are either not expressed or only weakly expressed in most cells. Restricting the analysis to 2,000-3,000 highly variable genes greatly reduces dimensionality, improves computational efficiency, and lowers memory usage.

⚙

Filtering Technical Noise

Single-cell sequencing data contain substantial stochastic noise. Small fluctuations in many lowly expressed genes mainly reflect technical noise, including dropout events where a gene is truly expressed but not detected. Removing these genes helps prevent algorithms from being misled by noise.

🔍

Enhancing Biological Signal

Housekeeping genes are stably and highly expressed across many cell types. Although their expression levels are high, they contribute little to meaningful biological variation and are not useful for distinguishing cell subtypes. Feature selection helps remove these highly expressed but uninformative genes.

3. Understanding the Main Algorithms

Once we understand the structure of the data, the next question is how modern single-cell toolkits solve this problem mathematically. This section summarizes three of the most widely used feature-selection strategies.

Variance Stabilizing Transformation (VST)

This is currently the default method in the R package Seurat. The core idea is to smooth the mean-variance relationship using a regression-based model.

  • ✓ 1. Compute log(mean) and log(variance) for all genes.
  • ✓ 2. Fit a trend line using Loess regression. This line represents the expected technical variance at a given expression level.
  • ✓ 3. Measure how far each gene’s observed variance deviates from the fitted trend, that is, compute the residual.
  • ✓ 4. Rank genes by residual size and select the top N genes, typically 2,000.

Dispersion-Based Method

This is the classic approach used in Scanpy and earlier versions of Seurat. It addresses mean-dependent variance through binning.

  • ✓ 1. Compute dispersion for each gene, where dispersion is variance divided by mean.
  • ✓ 2. To correct for the natural tendency of highly expressed genes to have higher dispersion, group genes into bins based on similar mean expression levels.
  • ✓ 3. Within each bin, standardize dispersion values by computing a Z-score.
  • ✓ 4. This binning strategy allows genes with unusually high relative variability to be identified fairly across low-, medium-, and high-expression ranges.

Pearson Residuals

This approach is typically associated with the SCTransform workflow and is widely considered one of the most statistically rigorous ways to account for sequencing depth effects.

  • ✓ 1. Fit a regularized negative binomial model to the count data.
  • ✓ 2. Use the model to estimate the expected count of each gene in each individual cell.
  • ✓ 3. Compute the Pearson residual between the observed and expected values.
  • ✓ 4. The variance of these residuals represents biological variability after accounting for sequencing depth. Genes with the largest residual variance are selected as HVGs.

💡 Practical Tips for Real Analyses

  • 1. Choosing the Number of Features (nfeatures)

    Selecting 2,000 genes is a common default. However, if your dataset has very high cellular heterogeneity, such as a whole-brain atlas or a multi-lineage developmental sample, increasing this to 3,000-5,000 often improves clustering resolution.

  • 2. Handling Batch Effects

    If your data include multiple batches, such as different sequencing runs, instruments, or collection days, feature selection can easily prioritize batch-associated genes. A strong recommendation is to identify HVGs separately within each batch, then combine them using either the intersection, which is more conservative, or the union, which preserves more information.

  • 3. Removing Confounding Gene Categories

    Mitochondrial genes (mt-) and ribosomal genes (RPS/RPL) are often identified as highly variable because of stress responses or metabolic state. If your main goal is cell-type identification rather than metabolic profiling, you should remove these genes from the HVG set before PCA.
